Grayslick Test Engine

Text Based Test Creation

Grayslick creates tests entirely from text. This allows for existing tests written in Office Word or Google Docs to be easily ported to Grayslick. It also allows for much faster test creation than a system like Google Forms. And since tests are just text files, sharing them with other teachers is easy.

Automatic Grading

Grading tests is an extremely robotic, time consuming task. While scantrons or similar systems are an improvement, students still need to wait days for their score. Grayslick can provide results immediately, for everything except essay and short response questions.

Essay and Short Response questions can not be automatically graded. Instead, they are put into a fast grading interface. This interface presents all responses on one page, and does not provide the student's name. This makes grading more streamlined, and removes any unintentional biases.

Protection Against Cheating

One of the largest problems with online tests is that students can look up the answers, or email answers to each other, or save the test and give it to other classes. Grayslick has several security features preventing this:
  • If the student leaves the tab, the test is submitted as is, and is flagged for cheating.
  • If the student leaves the browser window, the test is submitted as is, and is flagged for cheating.
  • Paste is disabled, so no notes can be copied than pasted into an essay question.
  • Copy is disabled, so tests can be distributed by students.
  • Developer tools are disables, so students can not turn off these protections.
  • There are several security suites, so browser addons or user scripts can not consistently disable protections.
  • The test will not render if the security is turned off.
  • Questions can be set to be randomly ordered, making cheating off of other student's screens impossible.
Additionally, using a phone to look up answers is far more subtle than using the computer, so even if a particularly clever script could disable security, there is no motivation for using one. Security is optional, and so can be disabled for surveys or labs.

Fast for students to take tests

Students do not need to create accounts. Instead, they simply visit a test url based off of your username and the test title. If you have toggled the test to be ready to take, it prompts them for their class code. This code splits students into different classes. Once they are in the test, they fill in their name, then complete the test. After it is completed, there score is given to them, and their answers recorded. After all your students have taken the test, you can release the answers, which will email all the students the test, the answers, and their responses, as a study tool. The student responses are then archived.

Tests use modern HTML5, not Adobe Flashplayer. This means students can take them using a computer, iPad, or phone.

Stoicism //option 2
Skepticism //option 3
Epicureanism //4
Neoplatonism //5
2 //this says that option 2 is the correct one

//this is how you embed a youtube video:
Watch the following video and write an essay on how the Greek Stoic philosophy is relevant in modern times.
media: https://www.youtube.com/watch?v=yu7n0XzqtfA

es: Write three to five paragraphs. //this is how you declare an essay question

Most questions follow this format, except the essay and short response question types:
question type: description

option
option

Any webpage can be embedded using the media: command. A variety of text formatting effects can be used. Questions can be set to be randomly ordered.
